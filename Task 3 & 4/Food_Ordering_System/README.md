## 📋 FOUR PHASES (4 WEEKS)

### Phase 1: Models & Scaffolding (Week 1)
**Points**: 50 | **Focus**: Foundation and CRUD
- Create MenuItem, Order, OrderItem models
- Set up database with proper relationships
- Generate CRUD scaffolding
- **Deliverable**: Functional but unstyled application

### Phase 2: Bootstrap Integration (Week 2)
**Points**: 60 | **Focus**: UI and responsive design
- Integrate Bootstrap framework
- Implement 12-column grid system
- Use Cards, Navbar, Forms, Tables
- Test on mobile, tablet, desktop
- **Deliverable**: Professional, responsive interface

### Phase 3: REST Principles (Week 3)
**Points**: 60 | **Focus**: API design and HTTP semantics
- Map UI actions to HTTP methods (GET, POST, PUT, DELETE)
- Implement statelessness (tokens in headers)
- Classify endpoints by idempotency
- Write comprehensive REST API documentation
- **Deliverable**: Well-documented REST API

### Phase 4: URI Design (Week 4)
**Points**: 50 | **Focus**: REST naming conventions
- **MUST USE**: Plural nouns only (/menu-items, /orders, /customers)
- **MUST AVOID**: Verbs in URIs (no /create, /delete, /get)
- **MUST IMPLEMENT**: Hierarchical resources (/orders/:id/items)
- Complete endpoint list
- **Deliverable**: RESTful API endpoints

### Bonus: Git & Documentation
**Points**: 10 | **Focus**: Professional workflow
- Meaningful commits and tags
- Professional README
- Code comments
- API documentation

## 🔍 CRITICAL RULES (MUST FOLLOW)

### REST URI Design (Phase 4)
```
✓ CORRECT:
GET /menu-items
POST /orders
PUT /orders/:id
DELETE /customers/:id
GET /orders/:id/items

✗ INCORRECT:
GET /menuitem (singular)
POST /createOrder (verb)
GET /getOrders (verb)
DELETE /order/:id/deleteItem (verb)
POST /orders/:id/addItem (verb)
```

### HTTP Methods (Phase 3)
```
GET     = Retrieve data (safe, idempotent)
POST    = Create new (NOT idempotent)
PUT     = Replace entire resource (idempotent)
DELETE  = Remove resource (idempotent)
```

### Responsive Design (Phase 2)
```
✓ Mobile First (320px min)
✓ Tablet Optimized (768px)
✓ Desktop Responsive (1200px+)
✓ Bootstrap grid system used
✓ No horizontal scroll on mobile
```
